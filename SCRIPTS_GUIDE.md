# 话术配置指南

## 📝 话术是需要客户填写吗？

**是的！** 话术需要根据客户的实际业务来配置。

### 默认配置 vs 客户定制

| 项目 | 默认配置 | 客户需要填写 |
|------|----------|--------------|
| 欢迎语 | ✅ 已配置通用欢迎语 | 可修改为品牌专属欢迎语 |
| 价格回复 | ✅ 已配置通用回复 | 需填写实际价格策略 |
| 联系方式 | ❌ 占位符 (400-XXX-XXXX) | ✅ **必须填写真实信息** |
| 快递物流 | ❌ 未配置 | ✅ **电商客户必填** |
| 发货时间 | ❌ 未配置 | ✅ **电商客户必填** |
| 退换货政策 | ❌ 未配置 | ✅ **电商客户必填** |
| 工作时间 | ✅ 已配置通用时间 | 可修改为实际时间 |

## 🔧 如何配置话术

### 步骤 1：编辑 config.json

```bash
vim config.json
```

### 步骤 2：在 autoReplies 中添加

```json
{
  "autoReplies": {
    // ... 现有规则 ...
    
    "shipping_delivery": {
      "keywords": ["快递", "物流", "几天到", "配送"],
      "reply": "您好！我们默认发中通快递...",
      "priority": 3
    },
    
    "shipping_time": {
      "keywords": ["发货", "今天发", "什么时候发"],
      "reply": "您好！我们的发货时间安排...",
      "priority": 3
    }
  }
}
```

### 步骤 3：测试话术

```bash
# 测试关键词匹配
node test.js --test-reply "我的快递几天能到"
node test.js --test-reply "今天发不发货"
```

## 📊 话术优先级说明

| Priority | 说明 | 示例 |
|----------|------|------|
| 1 | 最低优先级 | 欢迎语 |
| 2 | 中等优先级 | 价格、联系方式 |
| 3 | 较高优先级 | 快递、发货、订单 |
| 4 | 最高优先级 | 投诉、紧急问题 |

**匹配规则**：优先级高的先匹配，同优先级按配置顺序。

## 🎨 话术优化技巧

### 1. 使用 Emoji 增加亲和力

```
❌ "发货时间：24 小时"
✅ "⏰ 发货时间：24 小时内"
```

### 2. 分点说明更清晰

```
❌ "我们支持 7 天退货商品要未使用包装完好"
✅ "✅ 退货条件：\n- 商品未使用\n- 包装完好"
```

### 3. 提供明确指引

```
❌ "有问题联系客服"
✅ "如需帮助，请回复「人工」或点击菜单「联系客服」"
```

### 4. 设置合理预期

```
❌ "明天一定到"
✅ "一般 1-3 天到达，具体以快递配送为准"
```

## 🆘 话术不生效怎么办？

### 检查清单

- [ ] 关键词是否拼写正确
- [ ] 是否包含在 keywords 数组中
- [ ] priority 是否设置合理
- [ ] 是否重启了服务
- [ ] 配置文件是否是有效的 JSON

### Debug 步骤

```bash
# 1. 启用调试模式
export DEBUG=true
node index.js

# 2. 发送测试消息
curl -X POST http://localhost:8899/wecom/callback \
  -H "Content-Type: application/json" \
  -d '{"FromUserName":"test","MsgType":"text","Content":"快递几天到"}'

# 3. 查看日志
tail -f logs/wecom-helper.log | grep "匹配"
```
