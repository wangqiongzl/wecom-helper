## 🔧 话术配置方法

### 1. 选择行业模板

```bash
# 复制全行业模板
cp config.templates.json config.json
```

### 2. 启用对应话术

编辑 `config.json`，将对应行业的话术 `_enabled` 改为 `true`：

```json
"shipping_delivery": {
  "_enabled": true,  // 改为 true 启用
  "_industry": "电商",
  "keywords": ["快递", "物流"],
  "reply": "..."
}
```

### 3. 自定义内容

修改话术中的占位符：
- `XXX` → 实际内容
- `400-XXX-XXXX` → 真实电话
- `www.company.com` → 实际官网

### 4. 测试验证

```bash
# 运行测试
node test.js

# 测试特定话术
node index.js --test-reply "快递几天到"
```

## 📈 话术优化建议

### 1. 使用 Emoji 增加亲和力

```
❌ "发货时间：24 小时"
✅ "⏰ 发货时间：24 小时内"
```

### 2. 分点说明更清晰

```
❌ "我们支持 7 天退货商品要未使用包装完好"
✅ "✅ 退货条件：\n- 商品未使用\n- 包装完好"
```

### 3. 提供明确指引

```
❌ "有问题联系客服"
✅ "如需帮助，请回复「人工」或点击菜单「联系客服」"
```

### 4. 设置合理预期

```
❌ "明天一定到"
✅ "一般 1-3 天到达，具体以快递配送为准"
```

## 🆘 话术不生效排查

1. **检查关键词** - 是否包含在 keywords 数组中
2. **检查优先级** - priority 是否设置合理
3. **检查启用状态** - _enabled 是否为 true
4. **重启服务** - 修改配置后需重启
5. **查看日志** - `tail -f logs/wecom-helper.log`
